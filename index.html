1. Introdução

================================
| NÃO COMERCIALIZE ESSE SCRIPT |
================================

JANELA PERSONALIZADA 2.1, por Reynolds Costa

Essa biblioteca tem a finalidade de simular os comandos alert, cofirm e prompt, com uma estética melhorada, similar à do S.O. Windows, e com a possibilidade de inclusão de códigos HTML, trazendo, assim, a possibilidade de se trabalhar com páginas inteiras dentro de uma simples caixa de diálogo, usando apenas JavaScript puro.

É possível criar janelas inteiras que simulam as janelas do S.O. Windows sem a utilização de bibliotecas adicionais como jQuery ou Bootstrap.

2. Compatibilidade

Essa biblioteca permite o uso das bibliotecas previamente publicadas combinarTeclas, sendo requerida a sua versão 2.0, e windowsLikeMenu, sendo requerida a sua versão 1.1.

Note-se, contudo, que em nenhum dos casos dever-se-ão ser incluídos no projeto os arquivos terminados em %_func.js dessas bibliotecas, devendo janelaPersonalizada2.1_func.js o único arquivo a ser utilizado.

3. Instalação

Essa biblioteca se baseia em um arquivo *.css e três arquivos *.js, todos disponíveis no presente repositório nas pastas listadas no exemplo abaixo.

A sua instalação deve se parecer com isso:

Ver código

<link rel = "stylesheet" type = "text/css" href = "css/janelaPersonalizada2.1.css"/>

<script type = "text/javascript" language = "JavaScript" src = "js/janelaPersonalizada2.1_main_mini.js"></script>

<script type = "text/javascript" language = "JavaScript" src = "js/janelaPersonalizada2.1_pref.js"></script>

<script type = "text/javascript" language = "JavaScript" src = "js/janelaPersonalizada2.1_func.js"></script>

Não é necessário usar o arquivo janelaPersonalizada2.1_main.js, mais pesado que sua contraparte minificada, disponível nesse repositório apenas para consulta e contribuição.

4. Utilização

4.1. Objeto principal

O funcionamento dessa biblioteca se baseia em um objeto JSON principal JP_preferencias contido no arquivo janelaPersonalizada2.1_pref.js.

Quando não definidas, as propriedades das janelas criadas pela biblioteca herdarão seus valores desse objeto principal, portanto a alteração de seus valores deve ser feita com cuidado, pois terá impacto no projeto inteiro, e a alteração de suas chaves NUNCA deverá ser feita, pois isso provocará erros irreversíveis.

Considerando a importância desse objeto, é válido que o compreendamos antes de conhecermos as funções da biblioteca. Suas chaves são declaradas como segue. Para obter mais informações sobre cada uma, basta clicar sobre ela.

4.2. Propriedades

4.2.1. De escrita

Além do objeto JP_preferencias, o arquivo janelaPersonalizada2.1_pref.js também é o lar de duas propriedades: JP_mult e JP_pai. Observemos suas definições.

4.2.1.1. JP_mult

Booleano que, quando verdadeiro, permite que clicar sobre uma janela que esteja atrás na pilha de janelas tome a dianteira sobre suas irmãs. Quando falso, animará ou fechará a primeira janela (para mais detalhes, veja o atributo mouse do item 4.1) que estiver à frente.

Apesar de não ser uma constante, recomenda-se, para a facilitação no processo de debug, que a alteração dessa propriedade seja feita apenas uma vez e apenas em um lugar: o arquivo janelaPersonalizada2.1_pref.js.

4.2.1.2. JP_pai

String que contém o id do elemento que conterá e restringirá as janelas. Caso seja deixada em branco, as janelas poderão ser redimensionadas e movidas por todo o corpo do documento.

No presente documento, essa propriedade tem o valor "principal", que é o id da coluna desse texto, de modo que nenhuma janela pode ser movida para fora dela, isto é, entrar no menu.

4.2.2. De leitura

4.2.2.1. alt

O objeto JP possui também um atribuito chamado alt, que é um booleano que assume verdadeiro quando as janelas estão em modo alt, isto é, a tecla Alt foi pressionada e os botões que possuem o atributo letra foram sublinados.

4.2.2.2. lista

O objeto JP possui também um atribuito chamado lista, que é um vetor com todos os objetos de todas as janelas criadas, o que pode ser bastante útil para recuperar informações.

4.3. Funções

4.3.1. limparBotao

O objeto JP possui uma função chamada limparBotao(), que desmarca a seleção de todos os botões de todas as janelas e impede que o botão padrão seja selecionado por 400 milissegundos.

O uso dessa função é recomendado logo antes de se editar ou criar uma janela, quando se deseja que o botão padrão não seja selecionado.

Para mais detalhes sobre a seleção do botão padrão, veja o atributo botaoPadrao do item 4.1.

Mais detalhes sobre a criação e edição de janelas estão abaixo.

4.3.2. Módulos procedurais

4.3.2.1. Módulo criar

Todas as funções desse módulo retornam um objeto que pode ser usado para orientar a si mesmo nas funções dos outros módulos, comumente chamadas de forma procedural. Para mais detalhes, veja o item 4.3.3.

4.3.2.1.1. Função principal

A função
JP.criar.principal(obj)
é a mais importante desse módulo no quesito técnico, e, no quesito conceitual, é a mais importante de toda a biblioteca. Ela cria uma janela. Todas as funções do módulo criar a utilizam, motivo pelo qual a examinamos primeiro.
Parâmetro Tipo Descrição Obrigatório
obj objeto Objeto estruturado de acordo com o descrito no item 4.1. SIM

Embora sua declaração seja obrigatória, todas as chaves do parâmetro obj são opcionais. Ele pode receber, inclusive, um novo parâmetro, igualmente opcional, chamado id, conforme abaixo:

Ver código

{

id : "minha-janela", // STRING

conteudo : {

// CONTEÚDO DA JANELA

},

estilo : {

// ESTILO DA JANELA

},

config : {

// CONFIGURAÇÕES DA JANELA

}

}

Esse id será o identificador da janela no elemento HTML. Para evitar a construção de elementos com mesmo id, esse identificador é acrescido de um número sequencial. O código acima, por exemplo, se inserido como parâmetro da função mencionada e não houverem outras janelas com mesmo id, criará uma janela chamada "minha-janela1". A próxima janela criada com esse código se chamará "minha-janela2" e assim por diante. Se omitida, a chave id assumirá o valor "janela". Para obter o nome completo da janela, com o numeral, utilize o objeto retornado quando da sua criação. Para mais detalhes, veja o item 4.3.3.1.

4.3.2.1.2. Função alerta

A função
JP.criar.alerta(_conteudo, _titulo, callback),
mantida da versão anterior dessa biblioteca, cria uma janela de alerta. É análoga à função alert, porém com mais espaço para possibilidades.
Parâmetro Tipo Descrição Obrigatório
_conteudo string Conteúdo da janela.
Se não definido, assume "undefined". 		NÃO
_titulo string Título da janela.
Se não definido ou deixado em branco, assume "Aviso". 		NÃO
callback função Função que é executada ao se clicar em "Ok" ou fechar a janela. NÃO
Por definição, independentemente das preferências definidas no arquivo janelaPersonalizada2.1_pref.js, as janelas criadas pela função
JP.criar.alerta(_conteudo, _titulo, callback)
  • não podem ser e não iniciam como maximizadas;
  • possuem um botão fechar;
  • podem ser fechadas com o mouse, mas não com o teclado (ver atributo config.fechar do item 4.1);
  • possuem um botão Ok pré-selecionado que fecha a janela e executa a função do parâmetro callback, se declarado.

Para alterar essas propriedades, será necessário trocar na instalação o arquivo janelaPersonalizada2.1_main_mini.js por sua contraparte não-minificada, janelaPersonalizada2.1_main.js, e alterá-las manualmente, mas isso não é recomendado.

O botão ao lado cria uma janela de alerta:

Ver código

JP.criar.alerta("Olá, mundo!");

O botão ao lado cria um alerta que avisa no console quando for fechado:

Ver código

JP.criar.alerta("Olá, mundo!", "", function() {

console.log("Fui fechado");

}

4.3.2.4. Módulo recursos

Todas as funções dessse módulo retornam utilidades.

4.3.2.4.1. Função botaoPadrao

Retorna um objeto.

A função
JP.recursos.botaoPadrao()
retorna as propriedades do primeiro botão declarado no arquivo janelaPersonalizada2.1_pref.js. Isto é útil porque, como a declaração de todos os atributos é obrigatória ao se criar um botão, essa função poupa linhas caso somente uma propriedade (geralmente, o atributo funcao) seja alterada.
O botão ao lado utiliza
JP.recursos.botaoPadrao()
para atribuir uma função diferente ao botão da janela:
Ver código

let janExemplo;

function exemplo() {

let botao = JP.recursos.botaoPadrao();

/*

O botão dessa janela terá o atributo letra igual a -1,

o atributo descr igual a "" e o atributo ativo igual a true,

como definido no arquivo janelaPersonalizada2.1_pref.js.

*/

botao.funcao = function() {

const id = janExemplo.get.id(); // Obtém o id dessa janela para operações

JP.alterar.editar(id, {

estilo : {

posicao : {

vertical : "acima"

}

}

});

JP.alterar.botaoOnOff(id, 0); // Desativa o botão

};

janExemplo = JP.criar.principal({

conteudo : {

cabecalho : {

titulo : "Exemplo",

botoes : {

maximizar : false,

fechar : function() { }

}

},

corpo : "<p>Essa janela tem um botão baseado em um botão padrão<p>",

botoes : [

{

"Subir" : botao

}

]

}

}

}

4.3.3. Objeto das janelas

Como previamente mencionado, todas as funções do módulo criar (ver item 4.3.2.1) retornam um objeto. Esse objeto não possui propriedades públicas, mas tem seus próprios módulos que, por sua vez, chamam funções análogas às do objeto JP. Por fazerem referência à uma janela específica, essas funções análogas dispensam o uso do parâmetro id.

Note-se, ainda, que nenhuma dessas funções pode ser usada na mesma função que criou a janela. Nesses casos, dever-se-á utilizar as funções dos módulos procedurais (ver item 4.3.2).
